# 🏭 上架流水线操作手册 — 30品/天

> 目标：每天 30 个品，流水线作业，批处理填基础信息 + 逐个精修

## 核心理念

**逐个产品全流程模式（不要这样做）：**
```
产品1: 填标题 → 填属性 → 填SKU → 卡住 → 修 → 保存 → 产品2...
                                  ↕ 每次卡住都停
```

**流水线模式（推荐）：**
```
Phase 0: 1688 选品采集（批处理30个）
Phase 1: 全部移入待发布（批处理）
Phase 2: 基础配置批量填（批处理 — 30个一模一样）
Phase 3: 逐个精修（只有区别字段）
Phase 4: 保存 → 通知检查 → 发布
```

## 一天时间分配

| 时间段 | 用户 | Agent |
|:--:|:--|:--:|
| 09:00-09:30 | 1688 选30个品，插件采集 | — |
| 09:30-09:45 | — | Phase 1: 全移入待发布 |
| 09:45-10:00 | 花 5 分钟 hover 分类 | — |
| 10:00-10:30 | — | Phase 2: 批量填基础配置 |
| 10:30-11:30 | 说「产品N填xxx」 | Phase 3: 逐个精修 |
| 11:30-11:45 | 检查确认 | Phase 4: 保存+通知 |

全天：用户≈1小时 | Agent自动化≈2小时 = 30品

## 阶梯上量

- 第1天：跑3品磨合
- 第2天：跑10品
- 第3天：30品全速
