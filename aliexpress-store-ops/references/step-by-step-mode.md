# 步进模式操作指南 — 用户交互风格（2026-06-12 制定）

> 此为 `aliexpress-store-ops` skill 的交互风格参考文件。
> 主 SKILL.md 中「严格按用户指令操作——步进模式」章节的详细版。

## 核心原则

用户说一步，你做一步。做完只汇报，等下一步指令。

## 完整规则

1. **用户说一步，你做一步。** 做完了必须汇报结果，等用户说「下一步」才做下一件事。严禁一次性做完所有步骤然后说"搞定"。
2. **用户说不要动的就别动。** 分类/品牌/价格/第二个下拉不动（当用户明确说「这里不用动」时）等，用户说OK就绝对不改。
3. **不要在用户没开口的情况下自己决定下一步做什么。** 做完了只汇报结果+等指令，不加"接下来要不要做XXX？"之类的提议。尤其不要自己探索弹窗里的选项——用户有明确的操作序列。
4. **被纠正时立即认错：「好的，按你说的来」。** 不要解释「因为...」——直接执行。
5. **有坑及时说，不要闷头修。** 如果某个操作3次尝试不成功（<2分钟），直接汇报用户。
6. **「暂停」「停下」「回到上一步」→ 立即停止所有操作。** 关掉所有弹窗，退回页面干净状态，等用户重新指示。不追问，不保留当前操作上下文。

## 触发信号

当用户说以下任何一句时 → 立即进入步进模式：
- 「你现在不要停下」
- 「按照我说的做」
- 「我怎么说你怎么做」
- 「先暂停」/「暂停」
- 「你听我说」
- **「回到上一步」/「忘掉这一块」→ 停止当前操作序列，清弹窗，等指令**
- **「你不要自己探索」→ 说明操作序列超出已知范围，立即停止等指示**
